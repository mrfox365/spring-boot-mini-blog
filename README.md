# Spring Boot Mini-Blog

Простий, але повноцінний веб-додаток для ведення блогу, створений на базі Spring Boot. Проєкт реалізує безпечну реєстрацію користувачів із підтвердженням через email, систему профілів, а також можливість публікації записів та коментування.

## Основний функціонал
* **Безпечна авторизація:** Реєстрація, вхід, відновлення забутого пароля.
* **Email верифікація:** Відправка 6-значних кодів та посилань для скидання пароля за допомогою Spring Mail.
* **Управління профілем:** Можливість зміни нікнейму, пароля, додавання біографії та дати народження.
* **Соціальна взаємодія:** Створення записів (постів) у загальній стрічці та додавання коментарів у реальному часі.
* **Клієнтський роутинг:** Захист закритих сторінок на базі `localStorage` (Vanilla JS).

## Технологічний стек
* **Backend:** Java 17, Spring Boot 3, Spring Security, Spring Data JPA, Spring Mail.
* **Database:** PostgreSQL.
* **Frontend:** HTML5, CSS3, Vanilla JavaScript (Fetch API).
* **Інше:** Lombok, Maven, BCrypt для хешування паролів.

## Структура проєкту
Бекенд побудований за класичною тришаровою архітектурою:
* `config/` — налаштування безпеки (Spring Security).
* `controller/` — REST API ендпоінти для обробки запитів з фронтенду.
* `dto/` — об'єкти передачі даних для безпечного обміну інформацією між клієнтом та сервером.
* `entity/` — сутності бази даних (`User`, `VerificationCode`, `Post`, `Comment`).
* `repository/` — інтерфейси для взаємодії з PostgreSQL через Spring Data JPA.
* `service/` — основна бізнес-логіка додатку.
* `resources/static/` — файли фронтенду (`index.html`, `auth.html`, `reset-password.html`).

## Кроки для запуску

### 1. Підготовка бази даних
* Встановіть PostgreSQL.
* Створіть порожню базу даних із назвою `blog_db`.
* Таблиці створяться автоматично завдяки налаштуванню Hibernate (`ddl-auto: update`).

### 2. Налаштування змінних середовища
Для безпеки, проєкт не містить жорстко закодованих паролів. Перед запуском налаштуйте наступні змінні середовища (Environment Variables) у вашій IDE або терміналі:
* `DB_USERNAME` — ваш логін до PostgreSQL (наприклад, `postgres`).
* `DB_PASSWORD` — ваш пароль до PostgreSQL.
* `MAIL_USERNAME` — ваш Gmail (для відправки системних листів).
* `MAIL_PASSWORD` — ваш Google App Password (16 символів).

### 3. Запуск проєкту
Відкрийте проєкт у вашій улюбленій IDE (IntelliJ IDEA, Eclipse) та запустіть головний клас `ProjectApplication.java`.
Або скористайтеся Maven через термінал:
```bash
mvn spring-boot:run
